14차시: 객체지향 프로그래밍(OOP) 마스터하기
객체지향 프로그래밍(Object-Oriented Programming, OOP)은 데이터를 '객체(object)'로, 처리하는 행위를 '메서드(method)'로 묶어 프로그램을 설계하는 방식입니다. 이를 통해 코드의 재사용성을 높이고, 유지보수를 용이하게 만들 수 있습니다. 파이썬은 완벽한 객체지향 언어로, 클래스(Class)와 객체를 통해 OOP의 핵심 개념을 구현합니다.

1. 기초 파이썬 예시 코드 (5개)
클래스를 정의하고 객체를 생성하는 가장 기본적인 방법을 익힙니다.

1) 클래스(Class)와 객체(Object) 생성
클래스는 객체를 만들기 위한 '설계도'이며, 객체는 그 설계도를 바탕으로 만들어진 '실체(인스턴스)'입니다.

# '사람'을 표현하는 Person 클래스 정의
class Person:
    # 클래스의 속성 (Attribute)
    species = "호모 사피엔스"

    # 클래스의 행동 (Method)
    def say_hello(self):
        print("안녕하세요!")

# Person 클래스의 인스턴스(객체) 생성
p1 = Person()

# 객체의 속성과 메서드 사용
print(p1.species)     # 출력: 호모 사피엔스
p1.say_hello()        # 출력: 안녕하세요!
2) 생성자 메서드 __init__
객체가 생성될 때 자동으로 호출되는 특별한 메서드로, 객체의 초기 속성을 설정하는 데 사용됩니다.

class Car:
    # 객체 생성 시 brand와 model 값을 필수로 받음
    def __init__(self, brand, model):
        self.brand = brand  # 인스턴스 속성 'brand' 설정
        self.model = model  # 인스턴스 속성 'model' 설정
        print(f"{self.brand} {self.model} 차량이 생성되었습니다.")

# Car 클래스의 객체를 생성하며 초기값 전달
my_car = Car("현대", "쏘나타")
your_car = Car("기아", "K5")

print(f"내 차는 {my_car.brand}의 {my_car.model}입니다.")
3) 인스턴스 속성(Instance Attribute)
각 객체(인스턴스)가 독립적으로 가지는 고유한 데이터입니다. self.속성이름 형태로 정의합니다.

class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

book1 = Book("파이썬 정복", "이순신")
book2 = Book("자료구조 마스터", "세종대왕")

# 각 객체는 서로 다른 속성 값을 가짐
print(f"책1: {book1.title} (저자: {book1.author})")
print(f"책2: {book2.title} (저자: {book2.author})")
4) 인스턴스 메서드(Instance Method)
각 객체가 호출할 수 있는 함수로, 첫 번째 인자로 self를 받아 인스턴스 속성에 접근할 수 있습니다.

class Calculator:
    def __init__(self, initial_value=0):
        self.result = initial_value

    def add(self, number):
        self.result += number
        print(f"+{number} -> 현재 결과: {self.result}")

    def subtract(self, number):
        self.result -= number
        print(f"-{number} -> 현재 결과: {self.result}")

# 계산기 객체 생성
calc = Calculator()
calc.add(10)       # 출력: +10 -> 현재 결과: 10
calc.add(5)        # 출력: +5 -> 현재 결과: 15
calc.subtract(7)   # 출력: -7 -> 현재 결과: 8
5) 독립적인 객체
하나의 클래스에서 생성된 여러 객체는 서로 영향을 주지 않는 독립적인 존재입니다.

class Counter:
    def __init__(self):
        self.count = 0
    
    def click(self):
        self.count += 1

c1 = Counter()
c2 = Counter()

c1.click()
c1.click()
c2.click()

print(f"카운터 1의 값: {c1.count}") # 출력: 2
print(f"카운터 2의 값: {c2.count}") # 출력: 1
2. 응용 파이썬 예시 코드 (5개)
상속, 메서드 오버라이딩 등 객체지향의 핵심적인 개념을 활용합니다.

1) 상속(Inheritance)
부모 클래스의 모든 속성과 메서드를 자식 클래스가 물려받아 사용하는 것입니다. 코드 재사용성을 극대화합니다.

# 부모 클래스
class Animal:
    def __init__(self, name):
        self.name = name

    def eat(self):
        print(f"{self.name}이(가) 먹이를 먹습니다.")

# 자식 클래스 (Animal을 상속)
class Dog(Animal):
    def bark(self):
        print(f"{self.name}이(가) 멍멍!")

# Dog 클래스는 Animal의 속성과 메서드를 모두 가짐
my_dog = Dog("바둑이")
my_dog.eat()    # 부모 클래스의 메서드 호출
my_dog.bark()   # 자식 클래스에 정의된 메서드 호출
2) 메서드 오버라이딩(Method Overriding)
부모 클래스로부터 물려받은 메서드를 자식 클래스에서 재정의하는 것입니다.

class Animal:
    def speak(self):
        print("동물이 소리를 냅니다.")

class Cat(Animal):
    # 부모의 speak() 메서드를 재정의
    def speak(self):
        print("야옹~")

class Bird(Animal):
    # 부모의 speak() 메서드를 재정의
    def speak(self):
        print("짹짹!")

cat = Cat()
bird = Bird()

cat.speak()  # 출력: 야옹~
bird.speak() # 출력: 짹짹!
3) 클래스 변수(Class Variable)
클래스의 모든 인스턴스(객체)들이 공유하는 변수입니다.

class Circle:
    pi = 3.14159  # 클래스 변수 (모든 객체가 공유)

    def __init__(self, radius):
        self.radius = radius # 인스턴스 변수

    def get_area(self):
        return self.radius * self.radius * Circle.pi # 클래스 변수 사용

c1 = Circle(5)
c2 = Circle(10)

print(f"반지름 5인 원의 넓이: {c1.get_area()}")
print(f"반지름 10인 원의 넓이: {c2.get_area()}")
print(f"c1의 pi: {c1.pi}, c2의 pi: {c2.pi}") # 모든 객체가 같은 pi 값을 공유
4) 정보 은닉(Information Hiding)
객체의 중요한 속성을 외부에서 직접 수정하지 못하도록 막고, 정해진 메서드를 통해서만 접근하도록 하는 기법입니다. (파이썬에서는 관례적으로 속성 이름 앞에 _ 또는 __를 붙여 표현합니다.)

class BankAccount:
    def __init__(self, initial_balance):
        self._balance = initial_balance  # '_'로 시작하여 외부 접근 자제를 권고

    def deposit(self, amount):
        if amount > 0:
            self._balance += amount
            print(f"{amount}원 입금 완료. 현재 잔액: {self._balance}원")
        else:
            print("입금액은 0보다 커야 합니다.")

    def get_balance(self):
        return self._balance

account = BankAccount(10000)
# account._balance = -5000 # 이렇게 직접 접근 및 수정하는 것을 지양해야 함
account.deposit(5000)
print(f"최종 잔액 조회: {account.get_balance()}원")
5) 컴포지션(Composition, 구성)
한 클래스가 다른 클래스의 객체를 속성으로 가지는 것입니다. ('is-a' 관계인 상속과 달리 'has-a' 관계를 나타냅니다.)

class Engine:
    def start(self):
        print("엔진 시동을 겁니다.")

class Car:
    def __init__(self):
        self.engine = Engine()  # Car 클래스가 Engine 객체를 소유(has-a)

    def drive(self):
        self.engine.start()
        print("자동차가 주행을 시작합니다.")

my_car = Car()
my_car.drive()
3. 심화 파이썬 예시 코드 (5개)
매직 메서드, 데코레이터, 추상 클래스 등 파이썬 OOP의 고급 기능을 다룹니다.

1) 매직 메서드 __str__와 __repr__
객체를 print() 함수로 출력하거나, 인터프리터에서 표시할 때 사용될 문자열을 정의하는 특별한 메서드입니다.

__str__: 사용자가 보기 편한 형태로, print(obj) 호출 시 사용
__repr__: 개발자가 객체를 명확히 식별하기 위한 형태로, 그냥 obj 입력 시 사용
class Vector2D:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"2D 벡터 ({self.x}, {self.y})"
    
    def __repr__(self):
        return f"Vector2D(x={self.x}, y={self.y})"

v = Vector2D(3, 4)
print(v)          # __str__ 호출 -> 출력: 2D 벡터 (3, 4)
print(repr(v))    # __repr__ 호출 -> 출력: Vector2D(x=3, y=4)
2) @property 데코레이터 (Getter, Setter)
메서드를 속성처럼 호출할 수 있게 만들어, 속성 접근을 제어하고 계산된 속성을 만드는 데 사용됩니다.

class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def fahrenheit(self):
        # fahrenheit '속성'을 읽을 때 이 메서드가 호출됨
        return self._celsius * 9 / 5 + 32

    @fahrenheit.setter
    def fahrenheit(self, value):
        # fahrenheit '속성'에 값을 할당할 때 이 메서드가 호출됨
        self._celsius = (value - 32) * 5 / 9

temp = Temperature(25)
print(f"섭씨: {temp._celsius}°C, 화씨: {temp.fahrenheit}°F") # 메서드지만 () 없이 속성처럼 접근

temp.fahrenheit = 77 # 화씨 값을 설정하면
print(f"새로운 섭씨: {temp._celsius:.2f}°C") # 섭씨 값이 자동으로 계산됨
3) @staticmethod와 @classmethod
@staticmethod: self나 cls를 받지 않는, 클래스 내에 정의된 일반 함수. 클래스와 논리적으로 관련있지만, 인스턴스 상태와는 무관한 유틸리티 함수에 사용.
@classmethod: 첫 인자로 클래스 자체(cls)를 받는 메서드. 인스턴스를 생성하는 팩토리 메서드 등에 주로 사용.
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day

    @staticmethod
    def is_leap_year(year):
        # 인스턴스 정보(self)가 필요 없음
        return year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)

    @classmethod
    def from_string(cls, date_string):
        # 클래스(cls)를 이용해 새로운 인스턴스를 만들어 반환
        year, month, day = map(int, date_string.split('-'))
        return cls(year, month, day)

# 정적 메서드 호출
print(f"2024년은 윤년인가? {Date.is_leap_year(2024)}")

# 클래스 메서드를 이용한 객체 생성
d = Date.from_string("2024-12-25")
print(f"생성된 날짜: {d.year}년 {d.month}월 {d.day}일")
4) 다형성(Polymorphism)
서로 다른 클래스의 객체가 같은 이름의 메서드를 호출했을 때, 각 객체의 특성에 맞게 다른 동작을 하는 것을 의미합니다.

class Dog:
    def speak(self):
        return "멍멍!"

class Cat:
    def speak(self):
        return "야옹~"

class Sheep:
    def speak(self):
        return "매에~"

# 이 함수는 어떤 동물 객체가 오든 상관없이 동작함
def make_animal_sound(animal):
    print(animal.speak())

make_animal_sound(Dog())   # 출력: 멍멍!
make_animal_sound(Cat())   # 출력: 야옹~
make_animal_sound(Sheep()) # 출력: 매에~
5) 추상 기본 클래스(Abstract Base Class, ABC)
자식 클래스가 반드시 구현해야 하는 메서드를 명시하는 '설계 규약' 역할을 합니다. 추상 클래스 자체는 인스턴스화할 수 없습니다.

from abc import ABC, abstractmethod

# Shape는 추상 클래스
class Shape(ABC):
    @abstractmethod
    def area(self): # area는 추상 메서드
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    # 추상 메서드인 area()를 반드시 구현해야 함
    def area(self):
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    # area()를 구현하지 않으면 에러 발생
    def area(self):
        return 3.14 * self.radius * self.radius

# s = Shape() # 에러! 추상 클래스는 인스턴스화 불가

r = Rectangle(10, 5)
c = Circle(7)
print(f"사각형의 넓이: {r.area()}")
print(f"원의 넓이: {c.area()}")
