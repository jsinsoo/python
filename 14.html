14차시: 객체지향 프로그래밍(OOP) 마스터

데이터와 행동을 하나로 묶어 관리하는 현대 소프트웨어 설계의 정수를 배웁니다.

STEP 1. 핵심 개념

클래스(Class)와 객체(Object)

클래스는 무언가를 만들기 위한 '설계도'이고, 객체는 그 설계도로 만든 '실체'입니다.

Class
(설계도: 붕어빵 틀)
➔
Instance / Object
(실체: 팥 붕어빵, 슈크림 붕어빵)
class Person:
    # 생성자: 객체가 만들어질 때 초기 속성 부여
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print(f"안녕하세요, 저는 {self.name}이고 {self.age}살입니다.")

# 객체 생성 (인스턴스화)
p1 = Person("홍길동", 30)
p1.introduce()
STEP 2. 코드 재사용

상속 (Inheritance)

기존 클래스의 기능을 그대로 물려받아 새로운 기능을 추가하거나 기존 기능을 수정(오버라이딩)할 수 있습니다.

class Animal:
    def speak(self):
        print("소리를 냅니다.")

# Animal 클래스를 상속받는 Dog 클래스
class Dog(Animal):
    def speak(self): # 메서드 오버라이딩
        print("멍멍!")

my_dog = Dog()
my_dog.speak() # "멍멍!" 출력
💡 super() 활용: 자식 클래스에서 부모 클래스의 메서드를 호출하고 싶을 때는 super().메서드이름()을 사용합니다.
STEP 3. 고급 기술

다형성(Polymorphism)과 정보 은닉

같은 이름의 메서드가 객체에 따라 다르게 동작하게 하거나, 중요한 데이터를 외부에서 함부로 바꾸지 못하게 보호합니다.

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance # __를 붙여 프라이빗(Private) 변수로 설정

    def get_balance(self): # 접근 제어 (Getter)
        return self.__balance

# 다형성 예시
class Cat:
    def speak(self): return "야옹~"

class Sheep:
    def speak(self): return "매에~"

def make_sound(animal):
    print(animal.speak())

make_sound(Cat())   # 야옹~
make_sound(Sheep()) # 매에~
