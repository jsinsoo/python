Q 1
클래스(Class)는 객체(Object)를 만들기 위한 '설계도'이다.
O
X
💡 상세 해설

클래스는 객체를 생성하기 위한 청사진(설계도)이며, 이 설계도를 기반으로 메모리에 실제 탑재되어 살아 움직이는 실체를 객체 또는 인스턴스(Instance)라고 부릅니다.

class Car: # 설계도 정의 pass my_car = Car() # 실제 객체 생성 print(type(my_car))
<class '__main__.Car'>
Q 2
하나의 클래스로부터 생성된 모든 객체는 동일한 속성 값을 가진다.
O
X
💡 상세 해설

동일한 클래스로부터 만들어진 객체라 하더라도, 각 객체(인스턴스)는 고유한 메모리 공간을 할당받으므로 서로 다른 속성 값을 지닐 수 있습니다.

class Character: def __init__(self, name): self.name = name c1 = Character("전사") c2 = Character("마법사") print(c1.name, "&", c2.name)
전사 & 마법사
Q 3
상속(Inheritance)은 'has-a' 관계를 표현하기 위해 사용된다.
O
X
💡 상세 해설

상속은 'is-a' (~은 ~의 일종이다) 관계를 명시할 때 씁니다. 'has-a' (~은 ~을 가지고 있다) 관계는 클래스 내부 속성으로 다른 객체를 포함시키는 컴포지션(Composition) 기법을 사용합니다.

class Weapon: def attack(self): return "공격!" class Knight: def __init__(self): self.weapon = Weapon() # Knight 'has-a' Weapon k = Knight() print(k.weapon.attack())
공격!
Q 4
메서드를 정의할 때 첫 번째 매개변수인 self는 생략할 수 있다.
O
X
💡 상세 해설

일반적인 인스턴스 메서드를 선언할 때는 호출 시 해당 객체 자신을 자동으로 넘겨받도록 첫 번째 인자로 반드시 `self`를 작성해주어야 에러가 나지 않습니다.

class Robot: def fly(self): # self 생략 불가능 return "하늘을 납니다." r = Robot() print(r.fly())
하늘을 납니다.
Q 5
자식 클래스에서 부모 클래스와 동일한 이름의 메서드를 정의하는 것을 '메서드 오버로딩'이라고 한다.
O
X
💡 상세 해설

부모에게 물려받은 메서드를 자식 클래스에서 재정의하여 덮어쓰는 기법은 '메서드 오버라이딩(Overriding)'이라고 합니다. 오버로딩은 인자의 타입이나 개수가 다를 때 다르게 매핑되는 개념이며, 파이썬에서는 순수 기법으로 지원하지 않습니다.

class Father: def greet(self): return "안녕" class Son(Father): def greet(self): return "안녕하세요!" # 오버라이딩 print(Son().greet())
안녕하세요!
Q 6
__init__ 메서드는 객체가 소멸될 때 자동으로 호출된다.
O
X
💡 상세 해설

`__init__`은 객체가 생성 및 초기화될 때 실행되는 특수 메서드(생성자)입니다. 소멸할 때 작동하는 메서드는 `__del__`(소멸자)입니다.

class Test: def __init__(self): print("생성") def __del__(self): print("소멸") t = Test() del t
생성 소멸
Q 7
클래스 변수는 해당 클래스의 모든 인스턴스가 공유하는 변수이다.
O
X
💡 상세 해설

클래스 직하단에 선언된 클래스 변수는 인스턴스마다 각각 공간을 할당받는 것이 아니라, 전역처럼 해당 클래스로 파생된 모든 인스턴스가 공유하여 접근합니다.

class Counter: count = 0 # 클래스 변수 c1 = Counter() Counter.count += 1 print(c1.count)
1
Q 8
@staticmethod로 정의된 정적 메서드는 self 매개변수를 받지 않는다.
O
X
💡 상세 해설

정적 메서드(`@staticmethod`)는 인스턴스 속성이나 클래스 속성에 관여하지 않고 독립된 로직을 처리하므로 `self`나 `cls` 인자를 암묵적으로 받지 않습니다.

class Calculator: @staticmethod def multiply(a, b): return a * b print(Calculator.multiply(3, 4))
12
Q 9
추상 클래스(Abstract Class)는 인스턴스를 직접 생성할 수 있다.
O
X
💡 상세 해설

추상 클래스는 미구현된 추상 메서드를 지닌 일종의 가이드 클래스이므로 직접 객체(인스턴스)화하는 작업이 불가능하며, 반드시 상속을 구현한 자식을 통해서만 인스턴스를 생성할 수 있습니다.

from abc import ABC, abstractmethod class Base(ABC): @abstractmethod def step(self): pass try: b = Base() except TypeError as e: print("에러:", e)
에러: Can't instantiate abstract class Base with abstract method step
Q 10
다형성(Polymorphism)이란 같은 이름의 메서드 호출에 대해 객체마다 다르게 반응하는 특성을 말한다.
O
X
💡 상세 해설

다형성은 하나의 외형(메서드 이름)을 유지하면서도 실행의 대수(객체 인스턴스 종류)에 따라 각기 다르게 행동을 최적화하고 구현해내는 성질을 뜻합니다.

class Dog: def cry(self): return "멍멍" class Cat: def cry(self): return "야옹" for animal in [Dog(), Cat()]: print(animal.cry(), end=" ")
멍멍 야옹

Q 11
객체가 생성될 때 속성을 초기화하기 위해 사용하는 특별한 메서드의 이름은?
  • __main__
  • __start__
  • __init__
  • __new__
  • __create__
💡 상세 해설

`__init__`은 파이썬 클래스의 가장 대표적인 던더(Dunder) 메서드로, 인스턴스가 만들어질 때 진입하여 기본 상태 값을 세팅하는 데 사용됩니다.

class Member: def __init__(self, level): self.level = level m = Member("VIP") print(m.level)
VIP
Q 12
부모 클래스 Animal을 Dog 클래스가 상속받는 올바른 문법은?
  • class Dog is Animal:
  • class Dog extends Animal:
  • class Dog(Animal):
  • class Dog inherits Animal:
  • class Dog < Animal:

타 언어처럼 `extends` 키워드를 쓰지 않고, 파이썬에서는 자식 클래스 이름 뒤에 소괄호 `()`를 연 후 부모 클래스를 인자 형태로 기입합니다.

class Animal: def breathe(self): return "숨쉬기" class Dog(Animal): pass d = Dog() print(d.breathe())

클래스 내에서 인스턴스 자신을 가리키는 데 사용되는 관례적인 매개변수 이름은?
  • this
  • object
  • instance
  • me
  • self

자바나 자바스크립트의 `this`와 같은 역할을 하며, 파이썬 프로그래머 사이에서는 인스턴스 첫 인자로 `self`를 명명하는 것이 절대적인 규칙처럼 활용됩니다.

class Identity: def check_self(self): return self i = Identity() print(i == i.check_self())

Q 14
다음 중 정보 은닉(Information Hiding)의 주된 목적은?
  • 실행 속도 향상
  • 객체의 내부 상태 보호 및 무결성 유지
  • 코드 길이 단축
  • 메모리 사용량 감소
  • 외부 접근 허용

사용자가 내부 주요 데이터를 변조하거나 직접적인 접근을 가해 내부 비즈니스 로직과 데이터 무결성이 깨지는 현상을 차단하고 객체 내부를 보호하는 캡슐화가 주 목적입니다.

class Bank: def __init__(self): self.__money = 1000 # 프라이빗 설정 b = Bank() try: print(b.__money) except AttributeError as e: print("접근 불가 에러 발생")

Q 15
@property 데코레이터를 사용하는 가장 큰 이유는?
  • 메서드를 상수로 만들기 위해
  • 메서드를 비공개로 만들기 위해
  • 메서드를 속성(attribute)처럼 접근하게 하기 위해
  • 클래스에 종속시키기 위해
  • 전역 함수로 만들기 위해
💡 상세 해설

`@property`는 메서드가 특정 연산을 거쳐 반환하는 값을 일반 인스턴스 필드(변수)처럼 괄호 없이 단편적인 문법으로 우아하게 끌어 쓰도록 디자인할 때 사용됩니다.

class Box: def __init__(self, size): self.size = size @property def area(self): return self.size * self.size b = Box(4) print(b.area) # b.area() 가 아닌 일반 필드처럼 사용

Q 16
다음 코드의 출력 결과는?
class A: x=10 / class B(A): pass / class C(A): x=20 / print(A().x, B().x, C().x)
  • 10 10 10
  • 10 10 20
  • 10 20 20
  • 오류 발생
  • None
💡 상세 해설

`B` 클래스는 내부 공간에 `x`가 따로 마련되어 있지 않아 부모 클래스인 `A`로 흐름(MRO)이 상향되어 부모의 `x=10`을 빌려 쓰고, `C` 클래스는 자신만의 `x=20`을 네임스페이스에 선언하여 보관하기 때문입니다.

class A: x = 10 class B(A): pass class C(A): x = 20 print(A().x, B().x, C().x)
10 10 20
Q 17
@classmethod로 정의된 메서드가 첫 번째 인자로 받는 것은?
  • self (인스턴스)
  • cls (클래스 객체)
  • method (메서드)
  • None
  • 인자를 받지 않음
💡 상세 해설

`@classmethod` 데코레이터를 이용해 구현한 메서드는 개별 인스턴스가 아닌 클래스 설계 틀 자체를 핸들링하도록 설계되어 매개변수 1순위 자리에 클래스 자체를 명명하는 `cls`를 수신합니다.

class Hello: @classmethod def print_cls(cls): return cls.__name__ print(Hello.print_cls())
Hello
Q 18
다음 중 '다형성(Polymorphism)'을 가장 잘 설명한 것은?
  • 객체가 여러 타입을 가지는 것
  • 기능을 물려받는 것
  • 동일한 호출에 대해 객체 타입에 따라 다르게 동작하는 것
  • 내부 구현을 숨기는 것
  • 여러 객체를 생성하는 것
💡 상세 해설

다형성의 근본은 호출부 코드를 균일하게 다루면서도, 매핑 결합된 실체 객체가 무엇이냐에 따라 실제 결과물 행동이 다양하게 전개되는 것에 의미가 있습니다.

class Py: def run(self): return "파이썬" class Ja: def run(self): return "자바" def execute(lang): print(lang.run()) # 동일한 run 호출에 각기 다르게 대답함 execute(Py()) execute(Ja())
파이썬 자바

class Item: def __str__(self): return "포션" print(Item())

Q 20
s.find("x")와 s.index("x")의 가장 큰 차이점은?
  • 인덱스 값이 다르다
  • 반환 타입이 다르다
  • 속도 차이가 크다
  • 찾는 값이 없을 때의 처리 방식
  • 기능이 동일하다
💡 상세 해설

검색하려는 문자열 조각이 타겟 데이터 내에 전무할 시, `find()`는 런타임 중단 없이 평화롭게 `-1`을 돌려주는 반면 `index()`는 예기치 못한 크래시(`ValueError`)를 유발하는 에러 전파 방식의 차이가 있습니다.

text = "abc" print("find 결과:", text.find("z")) try: text.index("z") except ValueError: print("index 결과: ValueError 예외 감지")
find 결과: -1 index 결과: ValueError 예외 감지
